genomicranges.GenomicRanges module#

class genomicranges.GenomicRanges.GenomicRanges(data: Mapping[str, Sequence | Mapping], number_of_rows: int | None = None, row_names: Sequence | None = None, column_names: Sequence | None = None, metadata: Mapping | None = None)[source]#

Bases: BiocFrame

GenomicRanges provides functionality to represent and operate over genomic regions and annotations.

To construct a GenomicRanges object from a DataFrame must contain columns “seqnames”, “starts”, “ends” and “strand”. If “strand” column is not provided, “*” is used as the default value for each genomic interval.

Note: Intervals are inclusive on both ends.

Additionally, GenomicRanges may also contain Sequence Information (checkout SeqInfo) as part of its metadata. It contains for each sequence name (or chromosome) in the gene model, its length. Additionally, (checkout SeqInfo) might also contain information about the genome, and if its circular (is_circular) or not.

All columns other than “seqnames”, “starts”, “ends” and “strand” are considered metadata columns and can be accessed by mcols().

Note: The documentation for some of the methods are copied from the GenomicRanges R/Bioconductor package.

__getitem__(args: Sequence[str] | Tuple[Sequence, Sequence | None]) → GenomicRanges[source]#

Subset the object.

This operation returns a new object with the same type as caller. If you need to access specific rows or columns, use the row() or column() methods.

Usage:

# made up chromosome locations and ensembl ids.
obj = {
    "ensembl": ["ENS00001", "ENS00002", "ENS00002"],
    "symbol": ["MAP1A", "BIN1", "ESR1"],
    "ranges": BiocFrame({
        "chr": ["chr1", "chr2", "chr3"]
        "start": [1000, 1100, 5000],
        "end": [1100, 4000, 5500]
    ),
}
gr = GenomicRanges(obj)

# different ways to slice.
gr[0:2, 0:2]
gr[[0,2], [True, False, False]]
gr[<List of column names>]
Parameters:

args (SlicerArgTypes) –

A Tuple of slicer arguments to subset rows and columns. An element in args may be,

  • List of booleans, True to keep the row/column, False to remove.

    The length of the boolean vector must be the same as number of rows/columns.

  • List of integer positions along rows/columns to keep.

  • A slice object specifying the list of positions to keep.

  • A list of index names to keep. For rows, the object must contain unique

    row_names and for columns must contain unique column_names.

  • A scalar integer to subset either a single row or column position.

    Alternatively, you might want to use row() or column() methods.

  • A singular string to subset either a single row or column label/index.

    Alternatively, you might want to use row() or column() methods.

Raises:

  • ValueError – Too many slices provided.

  • TypeError – If provided args are not an expected type.

Returns:

The same type as caller with the subsetted rows and columns.

width can be negative, in which case the flanking region is reversed so that it represents a prefix or suffix of the range.

promoters(upstream: int = 2000, downstream: int = 200) → GenomicRanges[source]#

Extend intervals to promoter regions.

Generates promoter ranges relative to the transcription start site (TSS), where TSS is start(x). The promoter range is expanded around the TSS according to the upstream and downstream arguments. Upstream represents the number of nucleotides in the 5’ direction and downstream the number in the 3’ direction. The full range is defined as, (start(x) - upstream) to (start(x) + downstream - 1).

Parameters:

  • upstream (int, optional) – Number of positions to extend in the 5’ direction. Defaults to 2000.

  • downstream (int, optional) – Number of positions to extend in the 3’ direction. Defaults to 200.

Returns:

A new GenomicRanges object with ranges replaced by promoter regions.
